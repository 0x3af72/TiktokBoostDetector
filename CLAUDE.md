# CLAUDE.md

Documentation for the **TikTok Boost Detector** repository — a Chrome (MV3)
extension that marks TikTok posts Organic / Mixed / Paid from engagement shape.
This file is the project's persistent memory: decisions, conventions, the data
model we reverse-engineered, and gotchas.

> **IMPORTANT — keep this file current.** Whenever you make a meaningful change
> (new dependency, module, selector, threshold, env var, design decision, or a
> non-obvious gotcha), update `CLAUDE.md` in the same change. Treat an
> out-of-date `CLAUDE.md` as a bug. Prefer recording *why* over *what*.

## Overview

- **Repo:** https://github.com/0x3af72/TiktokBoostDetector.git
- **Purpose:** Decide whether a TikTok post's numbers are organic proof a format
  works, or just ad budget — before copying ("skeleton-stealing") the format.
- **Type:** Manifest V3 Chrome extension. No build step, no runtime deps. Tests
  run on plain Node. Icons generated by a Node script.

## The classification framework (src/classify.js)

**Binary classifier** (ORGANIC vs BOOSTED + confidence %); rewritten 2026-06-07,
replaces the 3-stage spec. Pure engagement math, no identity checks.
`classify(raw)` returns `{verdict, label, short, color, action, tagline,
confidence, score, signals[], viewScale, viewScaleLabel, viewsHidden, override,
promoLabel, undetermined, format, metrics}` — UI renders the signal table
color-coded (negative=organic green, positive=boosted red).

**Hard override:** `promoLabel` (mapped from `isAd`) → BOOSTED 99%. Skippable via
`classify(raw, {promoOverride:false})` — content.js passes this from the
`promoOverride` setting (popup toggle, default on). When off, an `isAd` post is
scored on S1–S5 only and flagged `overrideSuppressed`. Note: `isAd` is sourced from
TikTok's JSON `isAd` boolean (not the visual tag); its exact semantics (Spark vs
Promote, persistence after a campaign ends) are undocumented — hence the toggle.

**Signals** (each −2 organic … +2 boosted): S1 likes/views ×3 · S2 saves/views ×3
(video/slideshow bands) · S3 saves/likes ×2 · S4 likes/comments ×1 · S5
comments_alive ×1 (optional). `S = 3·S1+3·S2+2·S3+1·S4+1·S5` × view-scale
(≥100K ×1.0 · 10–100K ×0.7 · <10K ×0.4 · hidden: drop S1/S2, `2·S3+S4+S5` ×0.8).
S>0 BOOSTED · S<0 ORGANIC · S=0 → ORGANIC @50% (undetermined).
`confidence = min(95, 50 + 4·|S|)` (cap 80% hidden). Band tables live in
`BANDS`/`WEIGHTS` in classify.js.

**Key design points:**
- Saves carry half the weight (S2+S3 = 5/10) — hardest signal to fake.
- `comments_alive` (S5) is an EXPLICIT optional input — only scored when caller
  passes `commentsAlive` or real `topCommentLikes` (≥1% of likes). We do NOT proxy
  it from likes:comments (would double-count S4); otherwise S5 is skipped.
- `format`: slideshow if `imagePost` OR URL `/photo/`, else video.
- Confidence <60% → `undetermined`: UI greys the badge ("barely a guess").

Bands/weights are provisional; recalibrate on a 30–50 post labeled set (promo-tagged
vs clearly organic, in-niche). The Node tests assert every spec worked example with
formula-exact scores (kittyweight ORGANIC 95 / looksteacher ORGANIC 70 /
thefabstory BOOSTED 82 / myjourney BOOSTED 82 / glowupcat BOOSTED 99 override).
Spec example arithmetic has two harmless slips vs its own formula (kittyweight
−12 not −10; myjourney S3 +1 not +2 at 3.07%) — verdicts identical.

## TikTok data model (reverse-engineered, verified live 2026-06-07)

The crux: **profile grids do NOT expose per-post stats in the DOM** — thumbnails
show only a view count. Full stats arrive via API. Sources, most→least complete:

1. **`/api/post/item_list/`** (and `/api/item/`, `/recommend/`, …) fetch/XHR
   responses → `itemList[]` each with:
   - `stats` / `statsV2`: `playCount`(views), `diggCount`(likes),
     `commentCount`, `collectCount`(**saves**), `shareCount`.
   - `isAd` (boolean), `imagePost` (present ⇒ **slideshow**), `desc` (caption),
     `author.{uniqueId,nickname,verified}`.
   `interceptor.js` patches fetch+XHR at `document_start` and recursively harvests
   any node carrying a `stats`/`statsV2` block (endpoint-agnostic, depth-bounded).
2. **`__UNIVERSAL_DATA_FOR_REHYDRATION__`** `<script>` JSON. On a **video page**:
   `__DEFAULT_SCOPE__["webapp.video-detail"].itemInfo.itemStruct` (full stats +
   isAd + imagePost + author). On a **profile page** it has only
   `webapp.user-detail` — **no item list** (hence the API interception).
3. **Video-page action bar DOM** (fallback, no views): `data-e2e="like-count"`,
   `comment-count`, `favorite-count` (saves), `share-count`.

DOM anchors: grid items are `[data-e2e="user-post-item"]` containing
`a[href*="/video/<id>"]`. Video id from URL: `/video/(\d+)`.

MAIN↔ISOLATED worlds can't share JS scope, only `window.postMessage` — that's why
the interceptor (MAIN) forwards records to content.js (ISOLATED).

## Decisions & gotchas (newest first)

- 2026-06-07: **Binary classifier rewrite** (ORGANIC/BOOSTED + confidence %),
  replacing the 3-stage spec. `promoLabel`←`isAd` is a hard override. S5
  (comments_alive) only scored on explicit input (no likes:comments proxy →
  avoids double-counting S4). Badge now shows `ORG/BST n%`; undetermined (<60%)
  greyed.
- 2026-06-07: **Slideshows use `/photo/` URLs**, not `/video/` — the original
  `/video/`-only selectors silently skipped every slideshow. Content script now
  matches both (`POST_RE`, `POST_LINK_SEL`). A `/photo/` detail page has NO
  embedded itemStruct but DOES expose like/comment/favorite in the DOM action bar
  (no views) → directly-opened slideshow = REPLICATION_REQUIRED unless grid/API
  supplied views.
- 2026-06-07: **Complete detection rewrite** to the deterministic 3-stage spec
  (identity override → per-view ratio tests → verdict). Replaced the paid-points
  model. `isAd` now maps to `accountType='brand'` (Stage 1).
- 2026-06-07: **Widget-expand bug fix** — the MutationObserver retriggered scan on
  our own UI writes, rebuilding the widget every frame so header clicks never
  landed. Fixed with: sig-cache (rebuild only on data change), delegated toggle on
  the persistent container, and an observer filter ignoring our own nodes. Grid
  badges are now clickable → popover with the same breakdown.
- 2026-06-07: Low-sample posts are analyzed anyway (no more `?`), just flagged low
  confidence — per user request.
- 2026-06-07: Profile stats require fetch/XHR interception — not in DOM/embedded
  JSON. Content scripts use `world: MAIN` (interceptor) + `ISOLATED` (UI).
- 2026-06-07: Initialized repo; `.env` git-ignored via `.gitignore`.
